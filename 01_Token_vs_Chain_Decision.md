
# Choose Your Track: Token vs New Chain

## Track A — Token (Fastest to Market)
Use existing chains (Arbitrum, Base, BSC, etc.).
You get:
- instant security from Ethereum ecosystem
- wallets/exchanges/DEX access
- lower cost and time

## Track B — New Chain (Most Work)
You are building:
- nodes, networking, consensus
- state model and storage
- wallets, explorer, indexer
- ops/monitoring, testnet tooling

This package includes BOTH:
- A working scaffold for Track A
- A blueprint + skeleton for Track B
